# Spckit AI - 프로젝트 요약

## 📌 프로젝트 개요

**Spckit AI**는 Google Gemini 2.0 Flash와 RAG(Retrieval-Augmented Generation) 기술을 활용한 AI 기반 PC 부품 추천 시스템입니다. 사용자의 자연어 쿼리를 이해하고 135,000개 이상의 실제 부품 데이터베이스에서 최적의 조합을 추천합니다.

## 🎯 핵심 성과

### 1. RAG 시스템 완전 구현 ✅
- **데이터 규모**: 135,660개 PC 부품 레코드
- **벡터 DB**: 3,000개 문서 벡터화 완료
- **검색 정확도**: 50-62% 유사도 달성
- **응답 시간**: 5-10초 (검색 + AI 생성)

### 2. 프로젝트 구조 현대화 ✅
```
SpckitAI/
├── frontend/     # 사용자 인터페이스
├── backend/      # RAG 시스템
├── assets/       # 3D 모델 (예정)
└── backup/       # 이전 버전
```

### 3. 기술 스택 완성 ✅
- **Frontend**: Vite + Vanilla JS
- **Backend**: Python + FastAPI
- **AI**: Gemini 2.0 Flash + Embedding API
- **Database**: ChromaDB (벡터 저장소)
- **Package Manager**: npm + uv

## 🔍 시스템 아키텍처

### RAG 파이프라인
```
[사용자 쿼리]
    ↓
[임베딩 생성] (Gemini Embedding API)
    ↓
[벡터 검색] (ChromaDB - 3,000 문서)
    ↓
[Top-K 검색] (K=5, 유사도 ≥ 50%)
    ↓

    ↓

```

## 🛠 기술적 도전과 해결

### 1. SQL 파싱 문제
**문제**: MySQL 덤프가 `INSERT INTO table VALUES` 형식 (컬럼명 없음)

**해결**:
- 두 가지 INSERT 형식 지원
- MySQL 특수 주석 제거
- 중첩 괄호 처리 개선
- 컬럼 수 불일치 예외 처리

### 2. ChromaDB 메타데이터 오류
**문제**: None 값으로 인한 타입 오류

**해결**:
- 메타데이터 정제 로직 추가
- None 값 자동 필터링
- 타입 검증 (bool, int, float, str)
- 안전한 문자열 변환

### 3. 한글 인코딩 문제
**문제**: Windows 콘솔 CP949 인코딩

**해결**:
- 배치 파일에 `chcp 65001` 추가
- SQL 파일 읽기 시 `errors="ignore"`
- UTF-8 기본 인코딩 설정

### 4. uv 빌드 오류
**문제**: `hatchling`이 패키지를 찾지 못함

**해결**:
- `pyproject.toml`에 명시적 패키지 정의
- `[tool.hatch.build.targets.wheel]` 설정
- 패키지 리스트 추가
