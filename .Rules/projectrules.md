# 중요!!
- 모든 설명과 마크다운 문서는 "무조건"한국어(Korean)로 응답 및 작성한다.
- 지어낸 거짓정보(할루시네이션 정보)를 쓰지 않는다. 
- 외부 라이브러리나 API, 패키지를 사용할 때에는 "반드시" 가장 최근 공식 문서에 명시된 방식을 철저히 조사하여    
    1. 프로젝트 내 버전에 맞는 사용법 숙지 
    2. 입출력 데이터 형태 
    3. 사용 가능한 함수와 요청으로 받는 파라미터 정확한 명칭과 변수형 
    4. 의존 패키지, 다른 패키지등과의 호환성 
    항목을 먼저 학습 후 작업을 시작한다.

# 코드관련
- 선언하고자하는 변수, 함수, 클래스, 컨포넌를 생성하기 전에 반드시 동일한 기능을 하는기존의 것이 있는지, 동일 이름이 있는지 전수조사 후 작업에 진입한다. 
- 변수명은 명확하게 생성
- 에러가 날 수 있는 부분(결과를 쉽게 논리적으로 파악하기 힘든 만큼 복잡한 경우 또는 정보가 적어 생소한 패키지를 사용하는 경우을 지침함)은 미리 예외 처리 및 로그 처리 등으로 디버깅애 용이하게 한다.
- 모듈이나 클래스, 함수 등에는 입츌력 인자가 어떻게 되는지 주석으로 나타낸다.
- 출력문에(터미널, 로그, 화면 출력 등) 이모지를 출력하지 않도록 한다.

- 구체적인 설명
- 문서가 길어지면 링크 목차 개요를 통해 쉽게 볼 수 있게 설정


# 데이터 분석 관련
- 한번에 분석을 마무리 하지 않고 데이터 전처리 -> 탐색적 데이터 분석 -> 수리통계학적 접근의 고급 데이터 분석 -> 머신러닝&딥러닝 기법으로 접근의 고급 데이터 분석 -> 과정 및 결과 시각화 -> 인사이트 도출 단계를 가지고 이전 단계를 단계별로 구상하고 실행해 결과에 따라 다음 계획을 유동적으로 바꾸는 방식으로 작업한다.  
- 데이터 분석 작업을 할 시에는 최대한 다양하고 전문적인 접근을 시도함.
- 다양하고 전문적인 통계적 기법을 적극 활용하여 최대한 많은 인사이트를 낼 수 있도록 한다.
- 그 과정에서 통계적 오류가 없도록 하며, 그 과정에서 특정한 방법을 선택한 것에 대한 구체적인 근거를 남긴다.

# 프론트엔드 관련

- 메시 그라데이션과 산만한 차트 등 전형적인 AI 스타일을 피한다. 
- 의미없거나 추상적인 글자들을 배치하지 않는다. 
- 컴포넌트 명과 설명, css스타일은 항상 문서화로 남기고 코드 수정 시 문서도 함께 수정한다. 

# 백엔드 관련
- 사용자 요구를 밑바닥부터 구현하지 않고, 사용할 수 있는 전문 전용 패키지및 라이브러리,템플릿, 모듈, 외부 API등을 철저히 조사 후 적극적으로 활용한다.  
- 읜존성, 호환성을 철저하게 검토하고 관리한다.

- 운영체제나 환경에 구애받지 않고 협업자 온보딩, 빌드, 배포를 편리하게 할 수 있게 가이드와 자동화 파일을 철저히 설계한다. 

# 전문성 및 기본 태도
- 처음 프로젝트 작업 진행 시 단순 마크다운 문서 읽기 뿐 아닌 코드 단위로 현재 프로젝트의 진행 상황과 구현 상황을 분석하고 시작한다.
- 무에서 유를 창조하는것이 아닌 깃허브를 통해 다른 사용자들의 비슷한 목적의 결과물들을 먼저 최대한 조사하고 공부 후 참고하여 새부 로직 및 코드를 구성하거나 그대로 가지고와 적용하는 방식을 이용한다.  
- 하나의 코드 파일에 몇백, 몇천자의 코드를 한번에 넣지 않고, 비슷한 기능, 함수, 패키지, 클래스, 컴포넌트, API등을 파일을 분리시켜 모듈화 하여 관리한다.
- 폴더를 카테고리별로 여래개의 하위 폴더로 구조화 하여 쉽게 프로젝트를 관리, 보수, 협업할 수 있도록 한다.
- 동일한 오류가 계속되어 반복되면 잠시 한 숨을 돌리고 커피한 모금 하고 좀 더 멀리서 코드를 지켜보며 일단은 차분하게 생각하고 오류 수정을 시작한다.
